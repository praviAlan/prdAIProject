## **1\. Overview**

**Product Name:** Personal Portfolio Website  
 **Owner:** Alan Horvat  
 **Purpose:**  
 Create a clean, professional website to showcase 11 academic projects from this semester, demonstrating technical skills, design thinking, and problem-solving ability to recruiters, professors, and peers.

**Goals:**

* Present all 11 projects clearly and attractively  
* Highlight your skills and growth  
* Make it easy for recruiters to evaluate your work quickly  
* Serve as a long-term portfolio foundation

---

## **2\. Target Audience**

* **Primary:** Recruiters & hiring managers  
* **Secondary:** Professors / evaluators  
* **Tertiary:** Other students / collaborators

**User Needs:**

* Quickly understand your skills  
* Browse projects efficiently  
* See code, demos, and outcomes  
* Assess your thinking process

---

## **3\. Scope**

### **In Scope**

* Homepage  
* Projects listing (11 projects)  
* Individual project detail pages  
* About section  
* Contact section  
* Responsive design (desktop \+ mobile)

### **Out of Scope (for now)**

* Blog system  
* User accounts  
* Backend database (can be static)

---

## **4\. Key Features**

### **4.1 Homepage**

**Purpose:** First impression \+ navigation

**Content:**

* Name \+ role (e.g., “Computer Science Student”)  
* Short intro  
* Featured projects (2–4 highlighted)  
* Navigation menu

---

### **4.2 Projects Page**

**Purpose:** Display all 11 projects

**Requirements:**

* Grid or list layout  
* Each project card includes:  
  * Title  
  * Short description  
  * Tech stack  
  * Thumbnail/image  
* Click → opens project detail page

---

### **4.3 Project Detail Page**

**Purpose:** Deep dive into each project

**Each project should include:**

* Title  
* Overview (what it is)  
* Problem statement  
* Your solution  
* Technologies used  
* Screenshots / demo  
* Challenges & learnings  
* Links:  
  * GitHub repo  
  * Live demo (if available)

---

### **4.4 About Page**

**Content:**

* Short bio  
* Skills (languages, tools)  
* Education  
* Optional: resume download

---

### **4.5 Contact Section**

**Options:**

* Email  
* GitHub  
* LinkedIn

(Optional: contact form)

---

## **5\. Functional Requirements**

* Responsive design (mobile-first preferred)  
* Fast loading (optimize images)  
* Simple navigation (≤ 3 clicks to any project)  
* External links open safely  
* Clean URL structure:  
  * `/projects`  
  * `/projects/project-name`

---

## **6\. Non-Functional Requirements**

* Performance: load under 2–3 seconds  
* Accessibility: readable fonts, contrast, alt text  
* Maintainability: easy to add future projects  
* SEO basics: page titles, meta descriptions

---

## **7\. Design Requirements**

**Style:**

* Minimal and modern  
* Focus on readability  
* Consistent spacing and typography

**Suggested UI elements:**

* Cards for projects  
* Sticky navigation bar  
* Hero section on homepage

---

## **8\. Tech Stack (Suggested)**

**Frontend options:**

* HTML \+ CSS \+ JavaScript (simple)  
* OR React / Next.js (more scalable)

**Styling:**

* Tailwind CSS (fast and clean)  
* OR plain CSS

**Hosting:**

* GitHub Pages  
* Vercel / Netlify
